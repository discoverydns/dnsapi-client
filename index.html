Introduction

This project is part of the DiscoveryDNS solution. It is a client to the DiscoveryDNS Server REST API.

Download

The prefered way of using the client is via maven style dependency (which will automatically include required dependencies) as follows:

<dependency>
  <groupId>com.discoverydns.dnsapi</groupId>
  <artifactId>dnsapi-client</artifactId>
  <version>1.0.2</version>
</dependency>

Obtain the latest dnsapi-client library here: dnsapi-client v1.0.2 (sources) | javadoc))

Usage

Examples are provided in the com.discoverydns.dnsapiclient.example package. However in short you will need:

  • To setup SLF4J logging (we use logback's native binding) for log messages
  • Instatiate a DNSAPIClientFactory and use it to create a client
    • the client factory will require an implementation of the DNSAPIClientConfig interface to provide it with configuration parameters
    • it can be provided with an ObjectMapperFactory, SSLContextFactory & TransactionLogHandler or you can use the default implementations
    • you can also use a custom TransactionIDStratergy if you like, or the default one will be used
    • if you use the default providers you will need to provide implementations of there configuration interfaces
    One created you can then create a client. The client is thread safe, and one client can be shared by many threads, however it can only execute one outstanding command per http connection configured
  • then simply follow the examples for how to create and execute each command object
